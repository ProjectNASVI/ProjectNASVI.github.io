Here's a detailed explanation of how Project NASVI operates:

How Project NASVI Works: Smart Navigation Assistance System for the Visually Impaired​

Project NASVI is designed to assist visually impaired individuals by offering real-time, audio-based navigation and obstacle detection. The system combines computer vision, audio feedback, and real-time processing to provide an intuitive experience that guides the user through their surroundings.

Step-by-Step Breakdown of How NASVI Operates:​

1. Capturing Visual Data with the Camera Module​

  • The system begins with a Raspberry Pi Camera Module mounted at a suitable angle to capture the user’s immediate surroundings.
  • The camera continuously streams video to the Raspberry Pi, providing real-time visual input.
  • Wide Field of View: Positioned to capture a broad range, the camera helps identify both nearby and mid-range obstacles, improving situational awareness.

2. Processing Visual Data for Object and Path Detection​

  • Object Detection:
    • The visual data from the camera is processed by the Raspberry Pi using a pre-trained object detection model (e.g., YOLO or MobileNet).
    • The model recognizes and classifies objects such as people, vehicles, signposts, and obstacles on the path.
    • The system processes each video frame individually, so the object detection model must be optimized for real-time performance.
  • Path Detection and Obstacle Avoidance:
    • Path Detection: The system uses edge-detection algorithms to identify clear walking paths, guiding the user toward safe spaces.
    • Obstacle Detection: The system calculates the approximate distance of detected obstacles from the user, helping to avoid potential collisions by alerting them to close-range or fast-approaching obstacles.

3. Analyzing the Data and Preparing Audio Feedback​

  • Data Processing: Once an object or path is detected, the system immediately interprets the object's position and type (e.g., "car to the right," "clear path ahead").
  • Prioritizing Information: The system prioritizes urgent feedback, such as vehicles or obstacles directly in the user’s path, over non-essential objects to reduce cognitive load and ensure timely alerts.
  • Contextual Awareness: Based on detected objects, paths, and user positioning, the system can adjust the feedback to suit the environment (e.g., crowded areas may require more frequent feedback than open, quiet spaces).

4. Audio Feedback System (Real-Time Guidance)​

  • Text-to-Speech (TTS) Conversion:
    • Detected objects and navigation cues are converted to audio instructions via a text-to-speech (TTS) engine such as pyttsx3 (offline) or gTTS (online).
    • Example Feedback: “Clear path ahead,” “Vehicle approaching on the left,” “Crosswalk in 10 meters.”
  • Continuous Audio Output:
    • The earpiece continuously delivers these real-time audio instructions to the user, ensuring they are informed without needing to look at or interact with a screen.
  • Hands-Free Operation: With no physical interaction required, users can stay aware of their surroundings with both hands free, allowing for natural walking movements.

5. User Interaction and Navigation​

  • Receiving Navigation Cues: The user listens to the audio cues to make informed decisions about movement and avoid obstacles.
  • Dynamic Feedback: As the user moves, the system dynamically updates to reflect changing surroundings, offering relevant guidance (e.g., adjusting directions or warnings based on nearby objects).
  • Adapting to Environments: Whether indoors, outdoors, in crowded spaces, or on quiet paths, the system adjusts its feedback to match the complexity of the environment, allowing the user to navigate safely and confidently.

6. Power Supply and Portability​

  • Battery Pack: A portable battery powers the Raspberry Pi and camera, allowing the system to operate for several hours.
  • Lightweight Design: The components are compact and lightweight, enabling the user to comfortably carry the system throughout their journey.

Summary of the Workflow:​

  1. Camera captures the environment ➔
  2. Raspberry Pi processes visual data using object and path detection models ➔
  3. Object and path data analyzed to determine urgency and relevance ➔
  4. Audio feedback generated via TTS to guide the user ➔
  5. User receives audio instructions through an earpiece, enabling safe navigation.

Advantages of Project NASVI's Working Model:​

  • Immediate Real-Time Feedback: Helps users respond promptly to changes in their surroundings.
  • Non-Intrusive Assistance: Audio instructions provide guidance without hindering the user’s movement.
  • Cost-Efficiency: Operates on affordable hardware, making it accessible and easy to reproduce.

Project NASVI thus provides a practical, technology-driven solution for independent, safe navigation, empowering visually impaired individuals to explore their environments confidently.
